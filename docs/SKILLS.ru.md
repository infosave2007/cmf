# Навыки: рой специалистов в одном файле

*Языки: [English](SKILLS.md) · [中文](SKILLS.zh.md)*

Один `.cmf`-файл несёт общий бэкбон плюс любое число **навыков** — наборов
замещающих тензоров, привитых из настоящих дообученных чекпойнтов. Рантайм
читает тензоры навыка *вместо* тензоров бэкбона (никогда не складывает и не
собирает вторую модель), поэтому переключение специалистов бесплатно: тот же
mmap, та же память, другие веса по тем же адресам. Промпт можно направлять к
нужному навыку автоматически — файл сам знает, какой из его специалистов
подходит, без обученного гейта и внешнего роутера.

Всё ниже воспроизводится тремя командами на настоящих чекпойнтах с
Hugging Face; файлы промптов и оценок лежат в
[`docs/skills-demo/`](skills-demo/). Замерено на `cortiq` 0.3.4+ с
Qwen2.5-0.5B-Instruct (q8) в роли бэкбона.

## Демо: один файл, четыре личности

```sh
# 1. Бэкбон
cortiq convert --model Qwen/Qwen2.5-0.5B-Instruct --quant q8 --output swarm.cmf

# 2. Три навыка из настоящих дообучений той же базы
cortiq skill add swarm.cmf --from vindows/qwen2.5-0.5b-text-to-sql-merged \
  --id sql --name "SQL assistant" --prompts docs/skills-demo/prompts-sql.txt

cortiq skill add swarm.cmf --from Vikhrmodels/Vikhr-Qwen-2.5-0.5b-Instruct \
  --id ru --name "Русский ассистент" --prompts docs/skills-demo/prompts-ru.txt \
  --quality docs/skills-demo/eval-ru.txt

cortiq skill add swarm.cmf --from ewre324/ewre324-Thinker-Qwen2.5-0.5B-Instruct-Reasoning \
  --id thinker --name "Step-by-step verifier" --prompts docs/skills-demo/prompts-think.txt
```

`skill list` показывает, что теперь несёт файл:

```
3 skill(s):
  sql        SQL assistant            72 tensor(s), 314.3 MB, layers [0..23], routable
  ru         Русский ассистент        72 tensor(s), 314.3 MB, layers [0..23], routable
      quality: {"backbone":11.875,"overlaid":11.027,"metric":"ppl","file":"eval-ru.txt"}
  thinker    Step-by-step verifier    72 tensor(s), 314.3 MB, layers [0..23], routable
```

Один файл 1.45 ГБ вместо четырёх отдельных моделей по 479 МБ (1.9 ГБ),
один mmap, а переключение личностей ничего не стоит при загрузке — тензоры
навыка это обычные записи каталога, на которые рантайм указывает вместо
тензоров бэкбона.

## Что измеримо стало лучше

**Роутинг: 6/6 на промптах, которых роутер не видел.** Файл направляет
свежие промпты к нужному специалисту с 3–5× запасом по ошибке
(`E = ‖r−BBᵀr‖²/‖φ‖²`, меньше — ближе):

```
$ cortiq route swarm.cmf -p "Show me the SQL to find orders with no matching invoice."
  sql      E = 0.0016   ← победитель
  thinker  E = 0.0073
  ru       E = 0.0112

$ cortiq route swarm.cmf -p "Расскажи, как приготовить сырники из творога."
  ru       E = 0.0050   ← победитель
  thinker  E = 0.0168
  sql      E = 0.0204

$ cortiq route swarm.cmf -p "Verify step by step whether 91 is divisible by 7 and by 13."
  thinker  E = 0.0047   ← победитель
  sql      E = 0.0165
  ru       E = 0.0190
```

**`ru`: −7.1% перплексии на русской прозе**, записано в реестр самого файла
при запекании (бэкбон 11.88 → с навыком 11.03 на отложенном русском
тексте). Разница видна и в генерации: бэкбон выдаёт кольцевые пункты
(«Стоимость аренды обычно выше, чем лизинг… Стоимость лизинга обычно
ниже»), навык пишет связную структурированную прозу.

**`thinker`: проверяет, а не утверждает.** Спросите *«Is 17077 a prime
number?»* — бэкбон уверенно выдумывает ложную факторизацию
(`17077 = 3 × 5699`, что неверно: 17077 — простое); навык начинает
систематическую проверку делимости. Тот же файл, `--skill thinker`.

**`sql`: структура text-to-SQL.** Навык отвечает на вопросы по схеме
оконными функциями и CTE там, где бэкбон пишет плоские джойны.

Честно о метриках: перплексия — правильный гейт для навыка, сдвигающего
*распределение* домена (как `ru` на русском тексте). Для
инструкционных SFT-доноров (`sql`, `thinker`) выигрыш — в *поведении
генерации*, а не в правдоподобии текста: меряйте их A/B-генерациями или
точностью на задачах и не удивляйтесь, что PPL на рукописном корпусе
стоит на месте или чуть хуже. `--quality` записывает в реестр то, что вы
измерили; он никогда молча не «освящает» навык.

## Как пользоваться

```sh
cortiq run swarm.cmf -p "..."                      # бэкбон
cortiq run swarm.cmf -p "..." --skill thinker      # явный оверлей
cortiq route swarm.cmf -p "..."                    # оценить все навыки
cortiq explain swarm.cmf --prompt "..."            # роутинг + распределение первого токена
cortiq run swarm.cmf -p "..." --blend auto         # мягкая суперпозиция топ-2
cortiq run swarm.cmf -p "..." --route-dynamic --trace   # переключение по токенам
```

`--blend auto` смешивает топ-2 навыка с весами softmax(−E/T) — промпт с
проверкой арифметики на хорватском лёг между `ru` и `thinker` примерно
50/50. `--route-dynamic` переключает активный навык *посреди потока* по
мере смены контекста; с `--trace` это видно (смешанный русско-SQL промпт
переключился `thinker → sql` ровно на генерации `… WHERE o.id IS NULL`,
затем обратно на бэкбон для русского объяснения). Пороги защищены
гистерезисом; настраиваются через `CMF_ROUTE_EON` / `CMF_ROUTE_EOFF`
(E включения / выхода), если ваши навыки лежат ближе друг к другу.

## Создание навыка: что действительно важно

`cortiq skill add` прививает тензоры **донора** в контейнер бэкбона:

```sh
cortiq skill add <backbone.cmf> \
  --from <hf-репозиторий-или-локальная-папка>  # донор той же архитектуры
  --id <id> [--name "..."]        # идентичность в реестре
  --layers all|A-B|i,j,k          # какие слои (по умолчанию все)
  --tensors ffn|attn|all          # какие семейства (по умолчанию ffn)
  --prompts file.txt              # 8+ примеров промптов → подпространство роутинга
  --quality held-out.txt          # PPL-гейт, записывается в реестр
  --output out.cmf                # по умолчанию: перезапись на месте
```

Тензоры донора переквантуются кодировкой самого бэкбона (q8-бэкбон
получает q8-навыки); прививка битово честна: перезапекание навыка из
*собственного* исходного чекпойнта бэкбона меняет PPL ровно на +0.0%.

**Берите доноров-дообучения вашего же бэкбона.** Это единственное
правило, решающее исход. FFN-тензоры со-адаптированы с вниманием вокруг:

- **SFT / merged-LoRA той же базы — работает.** Все три донора демо —
  дообучения Qwen2.5-0.5B-Instruct; их FFN-прививки когерентны и
  поведенчески различимы.
- **Родственники после continued-pretrain — как частичная прививка не
  работают.** Мы измерили, чтобы вам не пришлось: прививка FFN (и даже
  FFN+attention) из Qwen2.5-**Coder**-0.5B-Instruct — те же формы, та же
  семья, но глубоко дообученный на код сиблинг — даёт PPL 238 против 2.6
  у бэкбона. Его веса со-адаптированы с нормами и эмбеддингами, которых
  навык не несёт. Если донор уехал так далеко — нужен отдельный файл
  модели, а не навык.

**Промпты роутинга** (`--prompts`): 8 коротких промптов, которые *звучат
как пользователи навыка*. По ним фитятся статистики φ(x) (среднее +
PCA-базис ранга 2 над усреднённым скрытым состоянием на 2/3 глубины) и
сохраняются в заголовок — ~4 КБ на навык, дальше файл маршрутизирует себя
сам. Навыки, замещающие не-FFN тензоры, исключаются из *динамического*
роутинга (статический `--skill` работает) — рантайм предупреждает об этом.

## Ресурсы

| Что | Стоимость |
|---|---|
| Скачивание донора | один HF-чекпойнт (~1 ГБ bf16 для 0.5B), кэш в `~/.cache/cortiq/hf` |
| Время запекания | минуты на ноутбуке: квантование привитых тензоров + перезапись файла |
| Рост файла | только привитые тензоры: 314 МБ на FFN-навык всех слоёв у q8 0.5B (байты бэкбона не меняются) |
| Память при запекании | ~размер бэкбона + один тензор донора (донор читается через mmap) |
| Цена навыка в рантайме | ноль, пока неактивен; активный навык = скорость бэкбона (те же формы, те же ядра) |
| Цена роутинга | один prefill-проход промпта до φ-слоя |

Для меньшего размера ограничьте слои (`--layers 12-23` — вдвое меньше) и
проверьте через `--quality`, выживает ли нужное вам поведение — честный
компромисс в одном флаге.

## Как это устроено внутри

- Тензоры навыка — обычные записи каталога с именами `skill.{id}.{tensor}`:
  mmap как всё остальное, подгрузка страниц по первому касанию. Хранение
  растёт как |бэкбон| + Σ|дельт| ([спека §9](CMF_V2_SPEC.ru.md)).
- Рантайм разрешает каждый тензор через индирекцию источника: запись
  бэкбона или `skill.{active}.{name}` — замещение, никогда не сложение;
  полная модель навыка никогда не собирается.
- Роутинг — recon-argmin по аффинным подпространствам: без обученного
  гейта и классификатора — выбор является *свойством файла*. `route`,
  `explain`, `--blend`, `--route-dynamic` читают одни и те же дескрипторы
  по ~4 КБ.
- Поле `quality` реестра — честный контракт (claim 16): что измерено, на
  чём и с каким результатом. `skill list` его показывает.
